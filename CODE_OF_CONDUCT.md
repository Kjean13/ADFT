# Code of Conduct

## Our commitment

This project aims to provide a respectful, technically rigorous, and useful space for contributors, users, and reviewers.

Participants are expected to:

- discuss ideas in good faith;
- challenge code and claims, not people;
- prefer evidence, tests, and reproducible examples;
- communicate clearly and respectfully;
- avoid harassment, insults, discrimination, or intimidation.

## Expected behavior

Examples of positive behavior:

- reporting bugs with minimal reproduction steps;
- proposing corrections with tests and documentation updates;
- questioning claims when the code or validation does not support them;
- helping keep the repository honest about what the tool really does.

Examples of unacceptable behavior:

- personal attacks or insulting language;
- discriminatory or harassing behavior;
- deliberate misrepresentation of validation results or product capabilities;
- publishing private security details before coordinated disclosure.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, commits, pull requests, or other contributions that violate this code of conduct.

Serious or repeated violations may lead to temporary or permanent exclusion from the project spaces.

## Reporting

For conduct concerns, contact the maintainer privately through the same channel used for security or project contact.
